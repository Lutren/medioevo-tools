# Ley Kernel de Orden, Criterio y Disciplina - 2026-05-06

Status: `ACTIVE_LOCAL_LAW_DRAFT`

Short name: `LEY_KERNEL_OCD_HUMANO`

OCD in this workspace means:

- Orden;
- Criterio;
- Disciplina.

It is an operational discipline, not a clinical label.

## Law

```text
El sistema no actua si no sabe que es, donde vive, quien lo posee, que evidencia
lo respalda, que gate lo permite y que frontera protege.
```

## Kernel Rules

1. Every object has a lane: `OPEN`, `COMMERCIAL`, `BOOKS_EDITORIAL`,
   `PRIVATE`, `ARCHIVE` or `UNKNOWN_REVIEW_REQUIRED`.
2. Every action has an owner.
3. Every owner declares `Reads`, `May Touch` and `Must Not Touch`.
4. Every claim has evidence or is downgraded.
5. Every risky action passes through ActionGate.
6. Every external action requires target-specific proof.
7. Every private item stays private unless a target-specific release changes it.
8. Every agent writes to COMMS or a documented handoff.
9. Every raw source needs ficha/hash before becoming canon.
10. Every release has falsifiers and rollback.

## Human Order Standard

A workspace is healthy when a future agent can answer:

- What is this?
- Is it public, commercial, editorial, private or archive?
- Who owns it?
- What can touch it?
- What evidence proves its state?
- What would falsify the claim?
- What is the next safe action?

If those answers are missing, the state is `REVIEW`, not `APPROVE`.

## GEODIA Rule

GEODIA work must not confuse research, simulation, OS/kernel evidence and public
claims.

- Kernel evidence stays evidence.
- Simulation stays simulation.
- Public claims stay low-claim until validated.
- Private GEODIA remains private unless extracted into a sanitized package.

## Wabi-Sabi Rule

Wabi-Sabi learns policy and common-sense patterns. It must not mutate model
weights, aliases, external accounts or private data.

Allowed:

- decision ledger;
- local policy;
- examples;
- test evidence;
- safe routing.

Blocked:

- model weight mutation;
- external action;
- publication;
- private data extraction;
- broad filesystem cleanup.

## CEREBRO Rule

CEREBRO is not a dump. It is a curated system.

Theory enters as:

- ficha;
- term;
- alias;
- claim status;
- falsifier;
- public/private boundary;
- runtime translation if useful.

## Publication Rule

We publish:

- methods;
- templates;
- schemas;
- tools;
- synthetic examples;
- low-claim explanations;
- curated research.

We retain:

- full private runtime;
- raw formulas until ready;
- prompts;
- thresholds;
- real datasets;
- private books/canon;
- RPG/TCG;
- account/session material;
- proprietary calibration.

## Agent Rule

No agent is "trusted" by identity. An agent is trusted only by:

- explicit ownership;
- narrow touch scope;
- evidence;
- ActionGate;
- COMMS trail;
- validation.

## Failure Mode

If clutter returns, the kernel response is not deletion. It is:

1. observe;
2. ficha;
3. classify;
4. decide owner;
5. create migration map if needed;
6. gate;
7. act;
8. verify;
9. log.

## Final Principle

Order is not decoration. Order is safety, memory, speed and patrimony.
