# Duat Geodia

## Overview

pnpm workspace monorepo. Duat Geodia is a dark archeopunk, local-first observational simulation and agent-lab. Visuals evoke handmade 1980s/90s computing: pixel borders, amber/green phosphor CRT, dither textures, BBS/Amiga/DOS aesthetics.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **Frontend**: React + Vite (artifacts/duat-geodia) — wouter routing, framer-motion, shadcn/ui, Tailwind v4
- **API framework**: Express 5 (artifacts/api-server)
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec (note: re-fix lib/api-zod/src/index.ts after running — orval generates a stale barrel)
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

### api-server (Express 5 — previewPath: /api)

Backend routes:
- `GET/POST /api/witness` — WitnessLog hash-chained events
- `GET /api/witness/recent` — last N events for live ticker
- `GET/POST /api/handoff` — handoff artifact CRUD
- `GET /api/handoff/:id` — specific handoff
- `GET/POST /api/genesis` — Duat Genesis configuration snapshots
- `GET/POST /api/simulation/snapshots` — simulation state snapshots
- `GET /api/simulation/snapshots/latest` — most recent snapshot

## DB Schema (PostgreSQL via Drizzle)

- `witness_events` — hash-chained WitnessLog entries
- `handoff_artifacts` — exported handoff briefs (estado, certeza, inferencia, incógnita, acción siguiente, blockers, fingerprint)
- `genesis_configs` — Duat Genesis configuration snapshots
- `simulation_snapshots` — synthetic simulation state snapshots (PSI metrics, EML weight, leading model)

## Falsifier Controls

Four negative controls applied to each model vs current PSI to compute differential verdicts:
- `shuffle_velocity` — PSI forced to extreme low-signal (R≈0.05, ε≈0.93)
- `permute_radius` — R and ε inverted (1−R, 1−ε)
- `flat_noise` — all PSI fields → 0.5 (flat prior)
- `mirror_curve` — all fields reflected (x → 1−x)

PASS = model score drops >0.12 on perturbed data. WARN = drop ≤0.12. FAIL = model scores same/higher. Results logged to WitnessLog as `physics_compare` event.

## Epistemic Labels Used

All synthetic outputs are labeled: `SYNTHETIC_ONLY`, `DEMO_ONLY`, `PROTOTIPO`, `SIMULACIÓN`, `REVIEW_REQUIRED`, `BLOCK`. Physics claims are blocked by RapidGuardian unless independently validated.
